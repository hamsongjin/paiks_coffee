# Kiosk UI Improvement

## 목적

모바일 키오스크 사용자 화면의 사용성을 높이고, 메뉴 탐색부터 옵션 선택, 장바구니 확인, 주문 완료까지의 흐름을 단순하고 명확하게 만든다.

핵심 원칙은 다음과 같다.

- 기능 로직은 유지한다.
- 정보는 빠르게 읽히게 한다.
- 누르는 영역은 크게 유지한다.
- 하단 고정 액션을 중심으로 화면을 구성한다.
- 선택 상태와 가격 상태를 즉시 구분할 수 있게 한다.

---

## 사용자 흐름

1. 사용자는 `/menu`에서 카테고리를 고른다.
2. 상품 카드에서 가격, 상태, 품절 여부를 빠르게 확인한다.
3. 상품 상세에서 옵션을 선택한다.
4. 하단 고정 영역에서 선택한 옵션, 추가 금액, 최종 금액을 확인한다.
5. 장바구니에 담은 뒤 하단 바 또는 장바구니 페이지로 이동한다.
6. 장바구니에서 수량을 조정하고 최종 금액을 확인한 뒤 주문한다.
7. 주문 완료 화면에서 주문 번호와 완료 상태를 확인한다.

흐름상 중요한 것은 "탐색"보다 "선택 확인"과 "즉시 주문 이동"이다. 따라서 각 화면은 선택 상태와 다음 행동이 눈에 먼저 들어와야 한다.

## 화면별 레이아웃

### 1. `/menu` 상품 리스트

- 상단: 브랜드명, 현재 화면명, 간단한 안내 문구
- 그 아래: 카테고리 탭
- 본문: 상품 카드 리스트
- 하단: 장바구니 고정 바

레이아웃 특성:

- 모바일 폭 기준 1열 리스트를 유지한다.
- 상품 카드에는 이미지, 상품명, 가격, 상태가 함께 보여야 한다.
- 품절 상품은 시각적으로 흐리되지만, 카드 자체의 구조는 유지한다.
- 하단 장바구니 바 때문에 마지막 카드가 가려지지 않도록 충분한 하단 여백을 둔다.

### 2. 상품 상세 옵션 선택

- 상단: 뒤로가기
- 상품 정보: 이미지, 상품명, 부가명, 가격, 품절 여부
- 옵션 영역: 그룹별 카드형 옵션 리스트
- 하단: 선택 옵션 요약 + 최종 금액 + 담기 버튼

레이아웃 특성:

- 옵션 그룹은 하나의 카드 섹션으로 묶는다.
- 그룹 제목 옆에 badge를 둬서 필수/선택 여부를 즉시 보이게 한다.
- 하단 요약 영역은 sticky로 유지하고, 요약 정보와 액션 버튼이 함께 보여야 한다.

### 3. 장바구니

- 상단: 뒤로가기 + 화면 제목 + 간단한 안내
- 본문: 장바구니 아이템 리스트
- 하단: 총 금액 + 주문하기 버튼

레이아웃 특성:

- 각 아이템은 이름, 옵션, 수량, 단가, 총액이 한 카드 안에서 읽혀야 한다.
- 수량 조절 버튼은 터치 중심으로 크게 유지한다.
- 하단 주문 영역은 항상 접근 가능한 위치에 둔다.

### 4. 주문 완료

- 상단 또는 본문: 완료 상태 아이콘/표식
- 본문: 주문 완료 메시지
- 주문 번호 카드
- 하단: 메뉴로 돌아가기 버튼

레이아웃 특성:

- 완료 메시지는 짧고 직접적이어야 한다.
- 주문 번호는 가장 눈에 띄는 숫자 정보로 보여야 한다.
- 다음 행동은 하나만 둔다.

### 5. 하단 고정 장바구니 바

- 좌측: 장바구니 상태와 총 상품 개수
- 중앙: 총 금액
- 우측: 장바구니 이동 버튼

레이아웃 특성:

- 장바구니가 비어 있으면 숨긴다.
- safe-area inset을 반영한다.
- 메뉴/상세 화면과 겹치지 않도록 하단 여백 정책을 일관되게 적용한다.

## 컬러 / 간격 / 버튼 규칙

### 컬러

- 배경: 흰색과 아주 연한 neutral 톤을 기본으로 한다.
- 본문: neutral-950 계열로 높은 가독성을 유지한다.
- 보조 정보: neutral-500 ~ neutral-400 사용.
- 선택 상태: neutral-950 배경 + 흰색 텍스트 또는 흰 배경 + 진한 border로 명확하게 구분한다.
- 경고 상태: red-50 / red-700 계열을 사용한다.

### 간격

- 화면 바깥 여백은 모바일 키오스크 기준으로 16px 전후를 기본으로 한다.
- 카드 내부는 16px 전후의 padding을 기본으로 한다.
- 옵션 목록, 장바구니 리스트, 요약 영역은 요소 간 간격을 넉넉하게 둔다.
- 하단 고정 영역이 있는 화면은 안전 여백을 별도로 확보한다.

### 버튼

- 최소 높이는 44px 이상으로 잡는다.
- 주요 CTA는 full-width 또는 거의 full-width로 만든다.
- 보조 액션은 pill 또는 rounded card 형태로 처리한다.
- 수량 조절 버튼은 40px 이상을 유지한다.
- 버튼 상태는 hover보다 active, disabled, selected 상태를 우선한다.

## 옵션 선택 UX

옵션 선택은 상품 상세의 핵심 작업이다. 따라서 아래 원칙을 따른다.

### 공통 원칙

- 옵션 그룹은 카드 단위로 분리한다.
- 그룹 제목 옆에 badge를 붙여 필수 여부를 즉시 보여준다.
- 선택된 옵션은 border와 background로 동시에 강조한다.
- 추가 금액은 옵션명과 같은 영역에서 바로 보이게 한다.
- 텍스트가 길어도 카드 높이가 갑자기 무너지지 않게 한다.

### Radio 옵션

- 단일 선택 그룹은 라디오 카드 스타일을 사용한다.
- 선택 시 카드 전체가 강조되어야 한다.
- 상태 아이콘은 왼쪽 또는 카드 시작점에 배치한다.
- 가격은 우측에서 바로 읽히도록 유지한다.

### Checkbox 옵션

- 복수 선택 그룹은 체크박스 카드 스타일을 사용한다.
- 선택 시 체크 표시와 함께 배경이 바뀌어야 한다.
- 여러 항목을 한눈에 비교할 수 있게 한다.

### Counter 옵션

- 수량 조절은 별도 row 또는 카드 내의 영역으로 분리한다.
- `-` / `+` 버튼은 키오스크 터치 기준으로 크게 만든다.
- 수량 0 상태와 1개 이상 상태가 분명히 구분돼야 한다.
- 최대 수량 정보는 보조 배지로 제공한다.

### 선택 요약

- 하단 sticky 영역에 현재 선택된 옵션을 요약한다.
- 선택한 옵션이 없을 때는 빈 상태 메시지를 보여준다.
- 선택 개수, 추가 금액, 최종 금액을 분리해서 보여준다.
- 담기 버튼은 요약 영역 바로 아래에 붙여서 행동 흐름을 끊지 않는다.

## 장바구니 UX

장바구니는 확인과 수정이 동시에 이루어지는 화면이므로 다음 기준을 따른다.

- 아이템별 이름과 가격이 가장 먼저 보여야 한다.
- 옵션은 보조 정보이지만 너무 작게 숨기지 않는다.
- 수량 조절은 카드 하단에서 바로 수행할 수 있어야 한다.
- 삭제는 눈에 띄되, 실수 클릭을 유발할 정도로 크지 않게 둔다.
- 총 금액과 주문 버튼은 항상 화면 하단에서 유지한다.

장바구니 빈 상태는 단순한 안내보다 다음 행동 버튼이 더 중요하다.

- 빈 상태 메시지
- 메뉴로 돌아가기 버튼

---

## 주문 완료 UX

주문 완료 화면의 목표는 안내를 끝내고 사용자가 다음 행동을 쉽게 선택하게 하는 것이다.

- 완료 상태를 가장 먼저 보여준다.
- 주문 번호는 카드 또는 블록으로 분리해 강조한다.
- 불필요한 설명은 최소화한다.
- 주요 버튼은 메뉴로 돌아가기 하나만 둔다.

주문 완료 화면은 확인용 화면이지 탐색 화면이 아니므로 정보 밀도를 낮게 유지한다.

---

## 구현 계획

문서를 기준으로 구현은 다음 순서로 진행한다.

1. 공통 레이아웃과 글로벌 타이포/배경 톤 정리
2. `/menu` 상품 카드와 카테고리 탭 정리
3. 상품 상세 옵션 선택 UI 정리
4. 장바구니 리스트와 하단 주문 영역 정리
5. 주문 완료 화면 정리
6. 하단 고정 장바구니 바와 safe-area 정리
7. lint/typecheck 및 실제 모바일 폭 검증

이 순서로 진행하면 공통 톤을 먼저 맞추고, 화면별 개선이 뒤섞이지 않는다.
